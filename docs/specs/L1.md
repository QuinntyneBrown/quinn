# L1 Requirements — Quinn: Local LLM Coding Agent

## L1-1: CLI Interface
The system shall provide an interactive command-line interface where users can enter prompts, receive streaming responses, and interact with a coding agent in a conversational REPL, similar to Claude Code, Codex, or GPT CLI tools.

## L1-2: Local LLM Integration
The system shall connect to Ollama for all LLM inference, supporting model selection, streaming token output, and function/tool calling. All inference happens locally — no data is sent to remote LLM services.

## L1-3: Tool System
The system shall provide an extensible tool framework with built-in tools for file operations (read, write, edit), shell command execution, file/content search (glob, grep), web fetching, and browser automation via Playwright. Tools are the mechanism by which the agent takes actions in the user's environment.

## L1-4: Agent Loop
The system shall implement a multi-turn agent loop that receives user input, sends it to the local LLM with tool definitions, parses tool calls from the LLM response, executes them, feeds results back to the LLM, and repeats until the LLM produces a final text response. The loop handles errors gracefully and manages conversation context.

## L1-5: Privacy and Security
The system shall ensure no data leaves the user's machine except when the user explicitly invokes network-capable tools (web fetch, Playwright). There is no telemetry, no analytics, and no remote LLM calls. All tool executions require user awareness.
